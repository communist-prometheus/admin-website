# QA Catalog — admin-website & public-website

Living document. Each row in the matrix tracks one flow; every new flow
MUST be added here with its coverage status before merge.

## How this file is maintained

- New user-facing flows: add a row under the matching section with a
  stable `U-###` story ID.
- New edge cases for an existing flow: add under "Edge cases" with a
  `E-###` ID linked to the parent story.
- Tests: use `// covers: U-###` / `// covers: E-###` comments so the
  coverage map can be regenerated by grep later.
- Status legend: ✅ covered, ⚠️ partial, ❌ gap (must get a ticket).

## Critical bugs uncovered while writing this doc

1. **CRLF parseFrontmatter wipe**: Files authored on Windows silently lost
   all metadata on save. Root cause + fix: `src/utils/frontmatter/parse.ts`
   + `decode-response.ts`. Regression tests: parse.test.ts, decode-response.test.ts.

2. **CreateContentDialog form reset race**: Parent errors swallowed,
   dialog blanked. Fix: reset moved to show-watch + error try/catch in
   `src/views/ContentView.vue`.

3. **Mock SW FS missing settings**: LanguagesEditor never rendered rows
   in mock mode because `src/sw/mock/all-entries.ts` didn't include
   `settingsEntries`. Fixed.

4. **LanguagesEditor testid collision**: Parent passed `data-testid`
   as fall-through attribute to `<LanguagesTable>`, overwriting
   `languages-table` on the root `<table>`. Fixed by wrapping in
   `<section data-testid>` in `src/components/LanguagesEditor/LanguagesEditor.vue`.

5. **Body consumed twice on retry**: `recoverAndRetry` passed the
   already-read request to the retry path, causing TypeError which the
   SW_FETCH wrapper serialized as `{error}` 500 → schema decode failure
   on client. Fixed by cloning the request before exec in
   `src/sw/handlers/route.ts`.

6. **`recoverAndRetry` return-without-await**: Inner retry rejections
   escaped the try/catch, resulting in text `Recovery failed` 503
   responses that clients couldn't parse as JSON. Fixed by awaiting.

7. **Admin-website CI ran no tests**: `deploy.yml` built and deployed
   without executing unit or E2E. Added `bun run test:unit --run` and
   `playwright test` gates.

8. **16 dependabot security alerts**: All closed via version bumps
   + transitive overrides. `eslint-plugin-comments@1.0.0` was a phantom
   dead dependency blocking peer resolution. Removed.
